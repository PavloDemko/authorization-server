/v1/oauth/token

/v1/oauth/token post

post

This is the OAuth 2.0 grant that highly trusted apps utilize in order to access an API. In this flow the end-user is asked to fill in credentials (username/password) typically using an interactive form in the user-agent (browser). This information is sent to the backend and from there to authorization server. It is therefore imperative that the client is absolutely trusted with this information. For client side applications and mobile apps we recommend using web flows instead.

This is the OAuth 2.0 grant that highly trusted apps utilize in order to access an API. In this flow the end-user is asked to fill in credentials (username/password) typically using an interactive form in the user-agent (browser).   • Response

Media type: application/json

Body

/v1/oauth/refresh

/v1/oauth/refresh post

post

Use this endpoint to refresh an access token, using the refresh token you got during authorization.

post /v1/oauth/refresh

Use this endpoint to refresh an access token, using the refresh token you got during authorization.

  • Request
  • Response

Body

Media type: application/json

Type: object

Properties
  • grant_type: required (refresh_token - default: refresh_token)

    Denotes the flow you are using. To refresh a token use "refresh_token".

  • client_id: required (string)

    Your application's Client ID.

  • client_secret: required (string)

    Your application's Client Secret.

  • refresh_token: required (string)

    The refresh token to use.

Example:

{
  "grant_type": "refresh_token",
  "client_id": "3QBfya9tLgZ0uuTZ20KiYnz4DY4B9O-k",
  "client_secret": "LveswyAl8MVrdASqR3JQt2n-cNzWDMezPwvLWKj3dGISYOVDgYeT-2Dj2-a_JShi",
  "refresh_token": "YOUR_REFRESH_TOKEN"
}

HTTP status code 200

Body

Media type: application/json

Type: object

Properties
  • access_token: required (string)
  • expires_in: required (integer - default: 8640 - minimum: 8640)

    This is a expiration date of acccess token in seconds

  • refresh_token: (string)
  • token_type: required (Bearer - default: Bearer)

Example:

{
  "access_token": "eyJz93a...k4laUWw",
  "token_type": "Bearer",
  "expires_in": 86400
}

/v1/oauth/revoke

/v1/oauth/revoke post

post

Since refersh tokens never expire, you need to have a way to invalidate them in case they are compromised or you no longer need them. You can do use using this endpoint.

post /v1/oauth/revoke

Since refersh tokens never expire, you need to have a way to invalidate them in case they are compromised or you no longer need them. You can do use using this endpoint.

  • Request
  • Response

Body

Media type: application/json

Type: object

Properties
  • token: required (string)

    The refresh token you want to revoke.

  • client_id: required (string)

    Your application's Client ID. The client should match the one the refresh token was issued for.

  • client_secret: required (string)

    Your application's Client Secret. Required for confidential clients.

Example:

{
  "client_id": "3QBfya9tLgZ0uuTZ20KiYnz4DY4B9O-k",
  "client_secret": "LveswyAl8MVrdASqR3JQt2n-cNzWDMezPwvLWKj3dGISYOVDgYeT-2Dj2-a_JShi",
  "token": "YOUR_REFRESH_TOKEN"
}

HTTP status code 200

Body

Media type: application/json

Type: object

Example:

{
  "refresh_token": "YOUR_REFRESH_TOKEN"
}

/v1/user_info

/v1/user_info get

get

Given the authorization server access token obtained during login, this endpoint returns a user's profile. This endpoint will work only if openid was granted as a scope for the access_token.
